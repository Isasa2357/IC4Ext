# IC4Ext v1.0.1

## Added

- HW trigger / SW trigger configuration helpers
- `softwareTrigger()` API for D3D11 / D3D12 camera interfaces
- `AcquisitionStartMode::Deferred`
- Typed acquisition lifecycle API:
  - `startAcquisition()`
  - `stopAcquisition()`
  - `isStreaming()`
  - `isAcquisitionActive()`
- D3D12 `TimestampNearest` synchronization with selectable timestamp source
- `HostReceived` synchronization for independent camera clock domains
- D3D12 multi-camera synchronized display sample
- Optional recording of the same composed frame that is displayed
- OpenCV multi-camera analysis sample
- Motion-region, mean-luminance, changed-pixel-ratio, and region-count overlays
- IC4 device / transport diagnostic sample
- Public version constants in `IC4Ext/Version.hpp`
- Project version loaded from the root `VERSION` file

## Deferred multi-camera startup

Multi-camera code can configure every stream before starting frame transfer.

```cpp
IC4Ext::CameraCaptureConfig config;
config.acquisitionStartMode = IC4Ext::AcquisitionStartMode::Deferred;

capture0.open(selector0, config, backend);
capture1.open(selector1, config, backend);

cameraThread0.start();
cameraThread1.start();

cameraThread0.startAcquisition();
cameraThread1.startAcquisition();
```

The IC4 backend uses `ic4::StreamSetupOption::DeferAcquisitionStart` and the typed methods call the corresponding `ic4::Grabber` lifecycle API.

## Validation status

Validated hardware configuration:

```text
Camera model         : DFK 33UX252 x2
Resolution           : 1536 x 1536
PixelFormat          : BayerRG8
Requested frame rate : 160 fps
OffsetX / OffsetY    : 236 / 0
Trigger mode         : Hardware
Trigger source       : Line1
Frame matching       : TimestampNearest
Timestamp source     : HostReceived
Tolerance            : 4,000,000 ns
```

A long-running hardware test exceeded 85,000 synchronized sets with zero read errors on both cameras. Synchronization drops occurred mainly during initial alignment and remained nearly constant afterward.

```text
free-run + HostReceived : available; validated with 10 ms tolerance
HW trigger + Line1      : recommended; validated with 4 ms tolerance
SW trigger              : experimental; future improvement required
```

SW trigger remains available for experiments, but is not recommended for synchronization-sensitive operation in v1.0.1.
